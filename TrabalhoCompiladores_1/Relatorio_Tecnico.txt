3. Descrição do Funcionamento e Manual de Execução

O software simula a primeira fase de um compilador (Análise Léxica) para uma linguagem procedural simplificada. Ele lê um arquivo de código-fonte em formato texto (.txt), varre os caracteres e utiliza expressões regulares para identificar e agrupar esses caracteres em tokens válidos. Os tokens reconhecidos são armazenados sequencialmente em uma estrutura de dados dinâmica (Lista Encadeada) e impressos no console ao final da execução. Caso encontre cadeias ou caracteres não pertencentes ao alfabeto da linguagem, o analisador acusa erro informando a linha e a coluna.  

Como executar o software com autonomia:   
1 - Certifique-se de ter o compilador GCC (MinGW) e o Flex instalados no seu ambiente Windows.
2 - Abra o terminal na pasta do projeto e gere o código C do analisador através do Flex:flex trab_1.l
3 - Compile o código C gerado utilizando o GCC:gcc lex.yy.c -o analisador.exe
4 - Execute o software passando o nome do arquivo de testes desejado como argumento:
./analisador.exe fonte1.txt (Para testar o arquivo sem erros )
./analisador.exe fonte2.txt (Para testar o arquivo com erros )

4. Descrição da Linguagem e Expressões Regulares

A linguagem desenvolvida é procedural, fortemente tipada e não orientada a objetos. Para realizar a análise léxica, foram definidas 14 classes distintas de tokens.  
As classes de caracteres base são:
DIGITO: [0-9]
LETRA: [a-zA-Z]

As expressões regulares criadas para os 14 tokens são:  
TK_RESERVADA: ("while"|"for"|"if"|"else"|"do"|"int"|"void"|"float"|"double"|"long"|"than"|"bool"|"char"|"string"|"return"|"break"|"struct"|"type"|"static"|"const"|"def"|"import"|"try"|"catch"|"throw")
TK_IDENTIFICADOR: {LETRA}({LETRA}|{DIGITO}|"_")*
TK_FLOAT: ("+"|"-")?{DIGITO}*"."{DIGITO}+
TK_INTEIRO: ("+"|"-")?{DIGITO}+
TK_STRING: "\""([^\"\n])*\""
TK_CHAR: "\'"({LETRA}|{DIGITO})"\'"
TK_OP_ARITMETICO: ("+"|"-"|"\\"|":"|"x"|"\."|"*"|"++"|"--")
TK_OP_RELACIONAL: ("=="|"!="|">"|"<"|">="|"<=")
TK_OP_LOGICO: ("&&"|"||"|"!")
TK_OP_ATRIBUICAO: ("+="|"-="|"*="|"/="|"=")
TK_PARENTESES: [\(\)]
TK_CHAVES: [\{\}]
TK_COLCHETES: [\[\]]
TK_PONTUACAO: (","|"."|";"|":")

5. Descrição dos Tipos de Erros Tratados

O analisador conta com uma estrutura de contagem interna de quebras de linha (\n) e tamanho de tokens (yyleng) para mapear posições exatas. Ele trata os seguintes erros léxicos:  
- Caracteres Inválidos (Desconhecidos): O sistema captura símbolos soltos que não pertencem ao alfabeto da linguagem (ex: @, $, #) através da regra genérica de fallback (.). Quando ativada, a regra emite um alerta formatado apontando a falha.  
- Strings Malformadas: O analisador acusa erro ao encontrar strings cuja aspas duplas de fechamento foram omitidas antes da quebra de linha.  
- Identificadores Malformados: A tentativa de declarar variáveis iniciando com números (ex: 123variavel) é capturada parcialmente, fazendo com que a análise isole o número como TK_INTEIRO e acuse erro nas partes subsequentes ou as processe incorretamente, expondo a falha léxica ao programador.

6. Documentação dos Principais Métodos

O código fonte C implementado no arquivo Lex contém as seguintes rotinas de apoio:  

- adicionar_token(const char* classe, const char* lexema): Função responsável por alocar dinamicamente a memória para um novo nó da Lista Encadeada (TokenNode), inserindo a classe do token e o valor exato lido (yytext) no final da estrutura.  
- imprimir_lista(): Função que percorre toda a Lista Encadeada armazenada na memória, formatando e imprimindo no console a tabela de tokens na exata ordem de aparecimento no código-fonte.
- yylex(): Função principal gerada pelo Flex. Ela é o autômato finito propriamente dito, que consome o arquivo de entrada caractere por caractere, casando as expressões regulares e executando as ações (como chamar o adicionar_token ou contabilizar colunas).  
- yywrap(): Função auxiliar exigida pelo Flex para sinalizar que a leitura do arquivo chegou ao fim (retorna 1).  
- main(int argc, char argv): Ponto de entrada do programa. Recebe o nome do arquivo fonte passado via terminal, aponta a leitura do Flex (yyin) para este arquivo, chama o yylex() para executar a análise e, ao final, aciona imprimir_lista(). 

7. Processo de Construção do Software

O software foi concebido de maneira modular utilizando as seguintes ferramentas:  

- Ferramenta Auxiliar: Gerador de analisadores léxicos Flex (versão compatível com Windows).  
- Compilador: GCC (fornecido via ambiente MinGW) para a compilação do arquivo lex.yy.c gerado pelo Flex.  
- IDE/Editor: Visual Studio Code (VS Code) com terminal integrado (PowerShell) para a edição das expressões regulares e depuração do código C associado.  
- Versionamento: Git e GitHub, utilizados para realizar os commits de segurança, registrar o avanço do projeto através de controle de versão local e remoto.
